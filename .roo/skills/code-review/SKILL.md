# 코드 리뷰 스킬

## 절차

사용자가 리뷰 대상 파일/디렉터리를 지정하면 아래 단계를 따른다.

### Step 1: 대상 파일 파악

1. 지정된 파일 또는 디렉터리의 모든 소스 파일을 확인한다.
2. 파일 수가 많으면 사용자에게 범위를 좁힐지 확인한다.

### Step 2: 정확성 검토

- 비즈니스 로직이 의도한 대로 동작하는가?
- 엣지 케이스 (null, undefined, 빈 배열, 경계값)가 처리되는가?
- 에러 처리가 누락된 곳은 없는가?
- 비동기 코드에서 경쟁 조건이 발생할 수 있는가?

### Step 3: 보안 검토

- 사용자 입력이 검증/살균(sanitize)되는가?
- SQL 인젝션, XSS, CSRF 등 OWASP Top 10 취약점이 있는가?
- 인증/인가 로직에 빈틈이 없는가?
- 민감 정보(API 키, 비밀번호)가 코드에 노출되지 않는가?

### Step 4: 성능 검토

- 불필요한 반복이나 중복 계산이 없는가?
- 대량 데이터에서 성능 문제가 예상되는가?
- 메모리 누수 가능성이 있는가?
- 캐싱이 필요한 지점이 있는가?

### Step 5: 가독성 및 유지보수성 검토

- 함수/변수 이름이 의도를 명확히 전달하는가?
- 프로젝트 코딩 표준을 따르는가?
- 복잡한 로직에 설명이 부족하지 않은가?
- 중복 코드가 있는가?

### Step 6: 결과 보고

이슈를 심각도별로 분류하여 보고한다:

```
## 코드 리뷰 결과

대상: [파일/디렉터리 경로]
총 N개 이슈 (Critical: N, Major: N, Minor: N, Suggestion: N)

### Critical
- [파일:줄] 설명 → 개선 방안

### Major
- [파일:줄] 설명 → 개선 방안

### Minor
- [파일:줄] 설명 → 개선 방안

### Suggestion
- [파일:줄] 설명 → 개선 방안

### 긍정적 사항
- 잘 작성된 부분도 언급하여 균형 잡힌 리뷰 제공
```
