## 구현 가이드라인

1. **기존 코드 이해 우선**: 변경 대상 코드를 먼저 읽고 이해한 후 수정한다.
2. **점진적 변경**: 한 번에 여러 파일을 대규모로 수정하지 않는다. 작은 단위로 변경하고 검증한다.
3. **빌드 검증**: 코드 수정 후 빌드와 타입 체크가 통과하는지 확인한다.
4. **타입 안전성**: `any` 타입 사용을 최소화한다. 필요시 `unknown`과 타입 가드를 사용한다.

## 코드 작성 원칙

- **단일 책임**: 하나의 함수/클래스는 하나의 역할만 수행한다.
- **의존성 주입**: 외부 의존성은 생성자 또는 함수 파라미터로 주입받는다.
- **불변성 선호**: 가능하면 `const`, `readonly`, 불변 자료구조를 사용한다.
- **명시적 반환 타입**: public 함수의 반환 타입을 명시한다.

## 금지 사항

<!-- 프로젝트에서 절대 허용하지 않는 패턴을 나열 -->

- `console.log`를 프로덕션 코드에 남기지 않는다 (로거를 사용한다)
- `@ts-ignore` / `@ts-expect-error` 없이 타입 오류를 해결한다
- 민감 정보(API 키, 비밀번호 등)를 코드에 하드코딩하지 않는다
- 순환 의존을 생성하지 않는다
